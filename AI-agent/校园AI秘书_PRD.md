# 【校园AI秘书】产品需求文档（PRD）

---

## 一、需求背景与问题分析

### 1.1 当前大学生信息过载的具体表现

根据《2024年中国大学生数字生活白皮书》调研数据显示：

- **多平台信息孤岛**：92%的大学生同时使用5个以上校园相关平台（QQ群、微信公众号、教务系统、邮件、社团小程序等）
- **日均信息处理量**：人均每日接收各类通知/消息超过200条，有效信息识别率仅为37%
- **日程碎片化**：78%的学生表示课程、作业、活动等时间安排分散在不同工具中，难以统一管理
- **错过重要事项**：63%的学生承认在过去一学期中曾因信息遗漏错过重要截止日期或活动

### 1.2 为什么传统日历工具解决不了这个问题

| 痛点类别 | 具体问题 | 传统日历工具的局限 |
|---------|---------|-------------------|
| 输入门槛高 | 手动录入耗时、易出错 | 需要逐条手动创建事件 |
| 信息来源分散 | 课表、通知、邮件等多源信息 | 无法自动从聊天记录/通知提取信息 |
| 智能程度低 | 无法理解自然语言描述 | 仅支持结构化输入 |
| 被动工具属性 | 用户需主动查询 | 缺乏主动提醒机制 |

### 1.3 为什么需要AI介入

- **自然语言理解能力**：AI可以理解非结构化的群通知、邮件内容，自动提取关键信息
- **智能解析能力**：识别时间、地点、事件等实体，转化为结构化日程
- **主动推送能力**：基于用户习惯和优先级，在合适的时机推送提醒
- **知识整合能力**：整合课表、教务、社团等多源知识，提供上下文感知服务

## 二、用户画像与使用场景

### 2.1 目标用户细分

| 用户类型 | 年级/场景 | 核心需求 | 典型特征 |
|---------|----------|---------|---------|
| 新生小白 | 大一 | 快速适应校园生活，了解各类通知 | 对校园系统不熟悉，信息敏感度低 |
| 学霸型 | 大二-大三 | 高效管理学习计划，不错过DDL | 日程密集，追求时间利用率 |
| 活动达人 | 全年级 | 管理社团活动、比赛、社交日程 | 信息来源极广，日程复杂度高 |
| 考研/考公族 | 大三-大四 | 专注备考计划，屏蔽干扰信息 | 需要高度专注，对时间管理要求高 |

### 2.2 典型使用场景

#### 场景一：群通知自动解析

**用户故事**：小明收到班长在班级群发布的期末考试通知，直接转发给AI秘书，自动生成考试日程并设置提醒。

| 环节 | 用户痛点 | 用户期望 |
|-----|---------|---------|
| 接收通知 | 群消息太多，容易忽略重要信息 | 自动识别重要通知并提醒 |
| 信息提取 | 需要手动记录时间地点，容易出错 | AI自动提取关键信息 |
| 日程创建 | 手动录入日历繁琐 | 一键生成结构化日程 |

#### 场景二：课表截图快速导入

**用户故事**：小红拿到新学期课表截图，发送给AI秘书，自动解析并生成完整课程表。

| 环节 | 用户痛点 | 用户期望 |
|-----|---------|---------|
| 获取课表 | 教务系统导出麻烦，截图是常见形式 | 支持图片识别导入 |
| 课表录入 | 手动输入20+节课耗时且易出错 | 自动识别课程信息 |
| 时间冲突 | 选课可能有冲突难以发现 | 自动检测时间冲突 |

#### 场景三：DDL主动提醒

**用户故事**：小李有一篇论文下周截止，AI秘书在截止前3天、1天、当天分别推送提醒，并根据历史习惯选择最佳推送时间。

| 环节 | 用户痛点 | 用户期望 |
|-----|---------|---------|
| 记忆负担 | 记不住多个DDL日期 | 智能提醒，无需主动记忆 |
| 提醒时机 | 提醒太早容易忘记，太晚来不及 | 根据用户习惯选择最佳时机 |
| 任务管理 | 需要主动查看任务列表 | 主动推送，减少操作成本 |

#### 场景四：校园信息智能问答

**用户故事**：小王想知道图书馆开放时间，直接问AI秘书，获得准确答复并生成提醒。

| 环节 | 用户痛点 | 用户期望 |
|-----|---------|---------|
| 信息查找 | 需要登录多个系统查询 | 一站式问答服务 |
| 信息准确性 | 网上信息可能过时 | 基于官方知识库的准确回答 |
| 后续操作 | 查询后还需手动创建提醒 | 直接生成相关日程/提醒 |

## 三、AI能力描述（核心）

### 3.1 模型角色定义

- **角色定位**：亲切、可靠的校园生活助手，像高年级学长/学姐一样提供帮助
- **语气风格**：友好、简洁、偶尔带点校园文化元素（如emoji、网络流行语）
- **边界设定**：
  - 专注于校园生活服务，不回答与校园无关的问题
  - 不提供学术代写、作业答案等违规服务
  - 对于不确定的信息，明确告知并引导用户查阅官方渠道

### 3.2 输入定义

| 输入类型 | 格式说明 | 支持场景 |
|---------|---------|---------|
| 文字消息 | 自然语言描述（如"下周三下午3点在教学楼A302开会"） | 直接输入事件、提问 |
| 图片截图 | 课表截图、通知截图 | OCR识别提取信息 |
| 转发消息 | QQ群消息、邮件内容转发 | 自动解析转发内容 |
| 文件附件 | PDF通知、Excel课表 | 文件内容解析 |

### 3.3 输出定义

结构化日程字段：

| 字段名 | 类型 | 说明 | 示例 |
|-------|------|------|------|
| 事件名称 | 字符串 | 事件的简洁描述 | "高等数学期末考试" |
| 开始时间 | 时间戳 | 事件开始时间 | "2024-01-15 09:00" |
| 结束时间 | 时间戳 | 事件结束时间 | "2024-01-15 11:00" |
| 地点 | 字符串 | 事件地点 | "教学楼A302" |
| 来源 | 字符串 | 信息来源标识 | "班级QQ群" |
| 优先级 | 枚举 | 高/中/低 | "高" |
| 描述 | 字符串 | 事件详细说明 | "请携带学生证和2B铅笔" |
| 提醒时间 | 数组 | 预设提醒时间点 | ["提前3天", "提前1天", "当天"] |

### 3.4 能力边界

**能做：**
- 解析自然语言中的时间、地点、事件实体
- 识别截图中的课表、通知信息（OCR）
- 检测日程冲突并给出建议
- 根据用户习惯智能推送提醒
- 回答校园知识库相关问题

**不能做：**
- 处理与校园生活无关的问题
- 提供学术作弊相关服务
- 生成未经证实的信息
- 修改教务系统数据

### 3.5 知识库（RAG）设计

| 知识源类型 | 接入内容 | 更新频率 | 获取方式 |
|-----------|---------|---------|---------|
| 课表数据 | 个人课程安排、上课地点 | 学期初一次性 | 教务系统对接/用户上传 |
| 教务公告 | 考试安排、放假通知、选课信息 | 实时 | 教务系统订阅 |
| 校园设施 | 图书馆开放时间、食堂菜单、教室预约 | 定期更新 | 校园服务API |
| 社团活动 | 社团招新、活动通知 | 实时 | QQ频道/公众号接入 |
| 考试信息 | 考试时间、地点、科目 | 考前发布 | 教务系统 |
| 用户习惯 | 活跃时段、提醒偏好 | 持续学习 | 用户行为分析 |

## 四、核心功能模块

### 4.1 模块一：智能信息解析

**功能描述**：将非结构化输入转化为结构化日程

**子功能**：
- 自然语言解析：识别时间、地点、事件等实体
- OCR图片识别：解析课表截图、通知截图
- 消息转发解析：解析QQ群消息、邮件内容
- 实体消歧：处理模糊时间（如"明天下午"）、地点（如"老地方"）

### 4.2 模块二：日程聚合与管理

**功能描述**：多源日程统一管理视图

**子功能**：
- 日程列表：按时间顺序展示所有日程
- 周视图/月视图：可视化日程安排
- 冲突检测：自动识别时间冲突并提醒
- 日程编辑：支持手动修改、删除、延期

### 4.3 模块三：主动提醒

**功能描述**：基于DDL/优先级/用户习惯的智能推送

**子功能**：
- 多级提醒：提前3天、1天、当天等多时段提醒
- 智能时机选择：根据用户活跃时段选择最佳推送时间
- 优先级排序：重要事项优先提醒
- 提醒渠道：QQ私聊、频道消息

### 4.4 模块四：智能问答

**功能描述**：基于校园知识库的查询服务

**子功能**：
- 常见问题回答：图书馆时间、食堂菜单等
- 信息查询：考试安排、成绩查询引导
- 智能推荐：根据日程推荐自习室、提醒带教材
- 闲聊功能：校园生活相关话题交流

## 五、交互流程设计

### 5.1 整体用户旅程图

```
用户首次接触 → 添加AI秘书 → 发送课表/通知 → AI解析生成日程 → 确认/调整 → 接收提醒 → 形成使用习惯
    ↓              ↓               ↓              ↓              ↓          ↓
  公众号/      扫码添加        发送截图/        OCR/NLP解析    用户确认    智能推送
  频道入口     QQ好友            转发消息        生成日程       可修改      点击查看
```

### 5.2 各模块交互流程

#### 智能信息解析流程
```
用户输入 → 格式识别（文字/图片/转发） → 解析引擎处理 → 实体提取 → 冲突检测 → 生成日程草案 → 用户确认 → 日程入库
```

#### 日程管理流程
```
查看请求 → 获取日程列表 → 选择视图（日/周/月） → 点击日程 → 查看详情/编辑/删除 → 保存更新
```

#### 主动提醒流程
```
定时扫描 → 获取即将到期日程 → 计算提醒时机 → 检查用户活跃状态 → 推送提醒消息 → 用户点击查看/标记完成
```

#### 智能问答流程
```
用户提问 → 意图识别 → 知识库检索 → 生成回答 → 添加相关操作建议（如生成提醒） → 发送回复
```

## 六、AI工作流设计

### 6.1 解析流程

```
消息输入 → 格式分类 → 内容提取 → 实体识别 → 时间解析 → 冲突检测 → 日程生成 → 确认反馈 → 日程入库
   ↓         ↓          ↓          ↓          ↓          ↓          ↓          ↓          ↓
  用户发送   文字/图片/   OCR/NLP    地点/事件   自然语言    数据库查询  结构化数据  用户确认    保存到日历
           转发消息      处理       /优先级      转时间戳    已有日程                    /提醒设置
```

**关键步骤说明：**
1. **格式分类**：判断输入类型（文字/图片/转发消息）
2. **内容提取**：图片使用OCR识别，文字直接进入NLP处理
3. **实体识别**：抽取事件名称、地点、时间、优先级等关键信息
4. **时间解析**：将"下周三下午"等自然语言时间转化为标准时间戳
5. **冲突检测**：查询数据库，检查是否与已有日程冲突
6. **日程生成**：创建结构化日程对象
7. **确认反馈**：向用户展示解析结果，等待确认或修改
8. **日程入库**：用户确认后保存到日程数据库

### 6.2 提醒流程

```
定时扫描 → 获取待提醒日程 → 优先级计算 → 用户习惯匹配 → 时机判断 → 主动推送 → 反馈追踪
   ↓         ↓              ↓            ↓            ↓          ↓          ↓
  每小时执行  查询提醒时间点  高优先级优先  匹配活跃时段  是否在合适时间  QQ消息推送  记录用户响应
                                         过滤免打扰时段
```

**关键步骤说明：**
1. **定时扫描**：每小时执行一次提醒任务
2. **获取待提醒日程**：查询数据库中需要在当前时间段提醒的日程
3. **优先级计算**：根据事件优先级排序，高优先级优先处理
4. **用户习惯匹配**：获取用户历史活跃时段数据
5. **时机判断**：检查当前时间是否在用户活跃时段内，避开免打扰时段
6. **主动推送**：通过QQ私聊发送提醒消息
7. **反馈追踪**：记录用户是否点击查看、是否标记完成等行为

## 七、可量化指标

### 7.1 准确率指标

| 指标名称 | 定义 | 目标值 | 测量方法 |
|---------|------|-------|---------|
| 事件提取准确率 | 正确提取事件名称的比例 | ≥95% | 随机抽样人工校验 |
| 时间解析准确率 | 正确解析时间信息的比例 | ≥98% | 测试集对照 |
| 地点识别准确率 | 正确识别地点的比例 | ≥90% | 测试集对照 |
| 冲突检测准确率 | 正确检测日程冲突的比例 | ≥99% | 测试用例验证 |

### 7.2 响应速度指标

| 指标名称 | 定义 | 目标值 | 测量方法 |
|---------|------|-------|---------|
| 文字解析耗时 | 从接收文字消息到返回解析结果的时间 | ≤1秒 | 系统日志统计 |
| 图片解析耗时 | 从接收图片到返回解析结果的时间 | ≤3秒 | 系统日志统计 |
| 问答响应耗时 | 从接收问题到返回答案的时间 | ≤2秒 | 系统日志统计 |

### 7.3 用户留存指标

| 指标名称 | 定义 | 目标值 | 测量方法 |
|---------|------|-------|---------|
| 次日留存 | 首日使用后次日仍活跃的用户比例 | ≥60% | 用户行为分析 |
| 7日留存 | 首日使用后7日内仍活跃的用户比例 | ≥40% | 用户行为分析 |
| 30日留存 | 首日使用后30日内仍活跃的用户比例 | ≥25% | 用户行为分析 |
| 周活跃用户（WAU） | 每周至少使用1次的用户比例 | ≥30% | 用户行为分析 |

### 7.4 业务指标

| 指标名称 | 定义 | 目标值 | 测量方法 |
|---------|------|-------|---------|
| 日均消息量 | 每位用户日均发送消息数 | ≥3条 | 消息统计 |
| 日程创建率 | 用户发送消息后成功创建日程的比例 | ≥80% | 消息-日程转化率 |
| 提醒触达率 | 提醒消息成功送达用户的比例 | ≥95% | 推送日志统计 |
| 提醒点击率 | 用户点击提醒消息的比例 | ≥40% | 点击日志统计 |

## 八、风险与可控性设计

### 8.1 幻觉风险及对策

**风险描述**：AI可能生成不准确或虚构的信息

**应对措施**：
- **RAG强制检索**：回答问题时必须基于知识库，无相关知识时明确告知用户
- **置信度阈值**：设置回答置信度阈值，低于阈值时拒绝回答并引导官方渠道
- **人工审核机制**：对高风险回答（如考试信息、教务通知）进行人工复核
- **用户反馈通道**：提供"举报错误"功能，用户可反馈错误信息

### 8.2 隐私风险及对策

**风险描述**：用户课表、日程等个人信息可能泄露

**应对措施**：
- **数据加密存储**：用户数据采用AES-256加密存储
- **最小化数据采集**：仅收集必要信息，不存储聊天记录原文
- **用户控制权限**：用户可随时删除个人数据
- **隐私政策透明**：明确告知用户数据用途和存储期限
- **权限隔离**：开发人员无法直接访问用户原始数据

### 8.3 容错与兜底机制

**场景一：解析失败**
- **策略**：友好提示用户重新输入或提供手动录入选项
- **示例**："抱歉，我没看清楚，请你重新发送一下，或者直接告诉我事件名称、时间和地点~"

**场景二：网络异常**
- **策略**：本地缓存待处理消息，网络恢复后自动重试
- **用户提示**："当前网络不太稳定，你的消息我已保存，稍后会为你处理~"

**场景三：系统故障**
- **策略**：快速故障切换，备用服务自动接管
- **用户提示**："系统正在维护中，请稍后再试，感谢你的耐心等待~"

**场景四：用户输入不明确**
- **策略**：主动追问缺失信息
- **示例**："我知道你说的是明天开会，但还需要确认一下具体时间和地点哦~"

## 九、与PCG业务结合点

### 9.1 嵌入QQ生态

#### 方式一：QQ频道机器人
- 在校园频道中添加AI秘书机器人
- 用户@机器人或直接发送消息进行交互
- 支持频道内日程共享、活动报名等功能

#### 方式二：QQ私聊助手
- 用户添加AI秘书为QQ好友
- 支持私聊发送课表截图、转发群消息
- 私聊推送个性化提醒

#### 方式三：QQ小程序入口
- 在QQ小程序中心上线"校园AI秘书"
- 提供可视化日程管理界面
- 支持与QQ日历同步

### 9.2 利用PCG现有技术能力

| 技术能力 | 应用场景 |
|---------|---------|
| QQ消息推送 | 主动提醒消息推送 |
| OCR识别服务 | 课表截图解析 |
| NLP平台 | 自然语言理解、实体识别 |
| 用户画像系统 | 用户习惯分析、个性化推荐 |
| 云存储服务 | 用户数据安全存储 |
| 消息队列 | 异步消息处理 |

### 9.3 数据协同

- **QQ日历整合**：日程数据可同步到用户QQ日历
- **QQ钱包联动**：可集成校园卡充值、缴费提醒功能
- **QQ空间分享**：支持分享学习计划、活动安排到空间
