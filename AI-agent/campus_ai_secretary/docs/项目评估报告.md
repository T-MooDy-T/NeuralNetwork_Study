# 校园 AI 秘书 - 项目评估报告

---

## 一、项目定位

### 1.1 核心定位
**「基于 AI 的校园智能助手」** - 为在校学生提供智能化的日程管理、信息推送和知识问答服务，通过自然语言交互和主动提醒，帮助学生高效管理学习和生活。

### 1.2 目标用户
| 用户群体 | 核心需求 |
|---------|---------|
| 在校大学生 | 日程管理、考试提醒、校园信息查询 |
| 研究生 | 科研日程、论文截止提醒、学术资源检索 |
| 教职工 | 会议安排、通知发布、信息整理 |
| 校园管理者 | 数据统计、系统监控、用户管理 |

### 1.3 价值主张
- **智能解析**：支持自然语言创建日程（如"明天下午3点开会"）
- **主动推送**：基于优先级和时间智能推送重要信息
- **知识问答**：语义检索校园知识库，快速获取答案
- **统一入口**：整合多渠道信息（QQ群、微信群、公众号）

## 二、功能概述

### 2.1 核心功能模块

| 模块 | 功能描述 | 状态 |
|------|---------|------|
| **智能解析** | 自然语言识别时间、地点、事件实体 | ✅ 已完成 |
| **日程管理** | CRUD操作、优先级管理、状态追踪 | ✅ 已完成 |
| **RAG知识库** | 语义检索、智能问答、来源追溯 | ✅ 已完成 |
| **主动提醒** | 多级提醒、定时扫描、避免重复 | ✅ 已完成 |
| **智能推送** | 多数据源聚合、优先级排序、弹窗提醒 | ✅ 已完成 |
| **用户认证** | JWT认证、角色权限管理 | ✅ 已完成 |
| **管理后台** | 可视化仪表盘、数据统计、系统管理 | ✅ 已完成 |

### 2.2 数据流架构

```
信息采集层 → 解析处理层 → 存储层 → 推送服务层 → 用户端
    ↓              ↓           ↓              ↓
  QQ/微信       LLM解析      MySQL        智能推送     Web后台
  公众号        实体识别     Qdrant       定时任务      消息弹窗
```

### 2.3 关键特性

1. **自然语言解析**：支持相对时间表达（明天、下周一等）
2. **多级提醒机制**：支持提前1天、3小时等灵活提醒
3. **智能优先级排序**：高优先级事件优先推送
4. **多渠道信息聚合**：QQ群、微信群、公众号统一管理
5. **模拟推送功能**：无需真实消息通道即可演示

## 三、技术架构

### 3.1 技术栈

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 后端框架 | FastAPI | >=0.104.0 | 高性能、异步支持、自动文档 |
| 前端框架 | Vue 3 | - | 响应式、轻量、生态成熟 |
| UI组件 | Element Plus | - | 丰富组件库、快速开发 |
| LLM | Qwen | - | 阿里云稳定服务、中文支持好 |
| 向量数据库 | Qdrant | >=1.7.0 | 内存模式适合演示、部署简单 |
| 关系数据库 | MySQL/SQLite | - | 生产用MySQL、开发用SQLite |
| 认证 | JWT | - | 无状态、跨平台 |
| 日志 | Loguru | >=0.7.0 | 结构化日志、自动轮转 |
